VERCEL DEPLOYMENT CLEANUP GUIDE
================================

PROBLEM:
- Multiple old deployments on Vercel (naar2noor2, naar2noor, naar222noor)
- Getting 404 NOT_FOUND errors
- Need to clean up and keep only naar-noor.vercel.app

SOLUTION:

1. GO TO VERCEL DASHBOARD
   - Visit: https://vercel.com/dashboard
   - Select your project "Naar-Noor"

2. DELETE OLD DEPLOYMENTS
   - Click on "Deployments" tab
   - Find all old deployments with names:
     * naar2noor2
     * naar2noor
     * naar222noor
   - Click on each one
   - Click "Delete" button
   - Confirm deletion

3. VERIFY CURRENT DEPLOYMENT
   - Should only see: naar-noor.vercel.app
   - Status should be: Ready
   - Domain should be: naar-noor.vercel.app

4. REDEPLOY CURRENT VERSION
   - Go to "Deployments" tab
   - Click "Redeploy" on the latest deployment
   - Wait for build to complete
   - Verify site loads at https://naar-noor.vercel.app

5. TEST THE SITE
   - Open https://naar-noor.vercel.app
   - Should load without 404 errors
   - All pages should work
   - Check console for any errors

WHAT WAS UPDATED:
- vercel.json: Added proper redirects and headers
- .npmrc: Configured for correct npm resolution
- package-lock.json: Regenerated with correct versions

AFTER CLEANUP:
- Only naar-noor.vercel.app will be active
- Old domains will no longer work
- All traffic goes to the correct deployment
- 404 errors should be resolved

MANUAL STEPS IN VERCEL UI:
1. Dashboard → Select Project
2. Settings → Domains
3. Remove any old domains
4. Keep only: naar-noor.vercel.app
5. Deployments → Delete old ones
6. Redeploy latest version

If still getting 404 after cleanup:
1. Clear browser cache (Ctrl+Shift+Delete)
2. Hard refresh (Ctrl+Shift+R)
3. Try incognito/private window
4. Check Vercel build logs for errors
5. Verify .npmrc is committed
6. Verify vercel.json is correct
