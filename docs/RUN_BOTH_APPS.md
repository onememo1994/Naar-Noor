# 🚀 Running Both Applications\n\nComplete guide to run the Naar & Noor frontend and backend simultaneously.\n\n## Prerequisites\n\nBefore starting, ensure you have:\n\n✅ **Node.js** 18.0.0 or higher\n✅ **npm** or **pnpm** (comes with Node.js)\n✅ **.NET SDK** 8.0 or higher\n✅ **SQL Server** (local or remote)\n✅ **Git**\n\n### Verify Installation\n\n```bash\n# Check Node.js\nnode --version\nnpm --version\n\n# Check .NET\ndotnet --version\n```\n\n---\n\n## Quick Start (5 Minutes)\n\n### Step 1: Clone Repository\n\n```bash\ngit clone https://github.com/Mostafa-SAID7/Naar-Noor.git\ncd Naar-Noor\n```\n\n### Step 2: Setup Backend (Terminal 1)\n\n```bash\ncd api-server\ndotnet restore\n```\n\nWait for restore to complete, then run:\n\n```bash\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj\n```\n\n**Expected Output:**\n```\ninfo: Microsoft.Hosting.Lifetime[14]\n      Now listening on: http://localhost:5000\ninfo: Microsoft.Hosting.Lifetime[0]\n      Application started. Press Ctrl+C to exit.\n```\n\n✅ **Backend is running at:** `http://localhost:5000`\n\n### Step 3: Setup Frontend (Terminal 2)\n\nOpen a new terminal window and run:\n\n```bash\ncd naar-noor\nnpm install\nnpm run dev\n```\n\n**Expected Output:**\n```\n✔ Compiled successfully.\n\n✔ Build at: 2026-05-26T12:00:00.000Z - Hash: abc123\n\nApplication bundle generation complete. [X.XXs]\n\n✔ Compiled successfully.\n\nWatch mode enabled. Watching for file changes...\n\n✔ Angular Live Development Server is listening on localhost:4200 ...\n```\n\n✅ **Frontend is running at:** `http://localhost:4200`\n\n---\n\n## Detailed Setup\n\n### Backend Setup (Detailed)\n\n#### 1. Navigate to Backend Directory\n\n```bash\ncd api-server\n```\n\n#### 2. Restore NuGet Packages\n\n```bash\ndotnet restore\n```\n\nThis downloads all required .NET dependencies.\n\n#### 3. Configure Database Connection\n\nCreate or update `src/NaarNoor.API/appsettings.Development.json`:\n\n```json\n{\n  \"ConnectionStrings\": {\n    \"DefaultConnection\": \"Server=.;Database=NaarNoor;Trusted_Connection=true;\"\n  },\n  \"Logging\": {\n    \"LogLevel\": {\n      \"Default\": \"Information\",\n      \"Microsoft\": \"Warning\"\n    }\n  }\n}\n```\n\n**For Remote SQL Server:**\n```json\n{\n  \"ConnectionStrings\": {\n    \"DefaultConnection\": \"Server=your-server.database.windows.net;Database=NaarNoor;User Id=admin;Password=YourPassword123!;\"\n  }\n}\n```\n\n#### 4. Create Database & Apply Migrations\n\n```bash\ndotnet ef database create --project src/NaarNoor.Infrastructure\n```\n\nOr apply existing migrations:\n\n```bash\ndotnet ef database update --project src/NaarNoor.Infrastructure\n```\n\n#### 5. Run Backend\n\n```bash\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj\n```\n\n**Verify Backend is Running:**\n\nOpen browser and go to:\n- API Health: `http://localhost:5000/health`\n- Swagger Docs: `http://localhost:5000/swagger`\n\n---\n\n### Frontend Setup (Detailed)\n\n#### 1. Navigate to Frontend Directory\n\n```bash\ncd naar-noor\n```\n\n#### 2. Install Dependencies\n\n```bash\nnpm install\n```\n\nThis installs all npm packages from `package.json`.\n\n#### 3. Configure API URL (Optional)\n\nIf backend is on different URL, update `src/app/services/api.service.ts`:\n\n```typescript\nprivate apiUrl = 'http://localhost:5000/api';\n```\n\n#### 4. Run Development Server\n\n```bash\nnpm run dev\n```\n\n**Verify Frontend is Running:**\n\nOpen browser and go to: `http://localhost:4200`\n\nYou should see the Naar & Noor homepage.\n\n---\n\n## Running Both Apps Simultaneously\n\n### Option 1: Two Terminal Windows (Recommended)\n\n**Terminal 1 - Backend:**\n```bash\ncd api-server\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj\n```\n\n**Terminal 2 - Frontend:**\n```bash\ncd naar-noor\nnpm run dev\n```\n\n### Option 2: Using npm-run-all (Single Terminal)\n\nInstall npm-run-all globally:\n\n```bash\nnpm install -g npm-run-all\n```\n\nCreate `package.json` in root directory:\n\n```json\n{\n  \"name\": \"naar-noor-workspace\",\n  \"scripts\": {\n    \"dev:backend\": \"cd api-server && dotnet run --project src/NaarNoor.API/NaarNoor.API.csproj\",\n    \"dev:frontend\": \"cd naar-noor && npm run dev\",\n    \"dev\": \"npm-run-all --parallel dev:backend dev:frontend\"\n  }\n}\n```\n\nRun both apps:\n\n```bash\nnpm run dev\n```\n\n### Option 3: Using Docker Compose\n\nCreate `docker-compose.yml` in root:\n\n```yaml\nversion: '3.8'\n\nservices:\n  api:\n    build: ./api-server\n    ports:\n      - \"5000:5000\"\n    environment:\n      - ConnectionStrings__DefaultConnection=Server=db;Database=NaarNoor;User Id=sa;Password=YourPassword123!;\n    depends_on:\n      - db\n\n  db:\n    image: mcr.microsoft.com/mssql/server:2022-latest\n    environment:\n      - ACCEPT_EULA=Y\n      - SA_PASSWORD=YourPassword123!\n    ports:\n      - \"1433:1433\"\n    volumes:\n      - sqldata:/var/opt/mssql\n\n  frontend:\n    build: ./naar-noor\n    ports:\n      - \"80:80\"\n    depends_on:\n      - api\n\nvolumes:\n  sqldata:\n```\n\nRun with Docker:\n\n```bash\ndocker-compose up\n```\n\n---\n\n## Verification Checklist\n\n### Backend Running?\n\n- [ ] Terminal shows \"Now listening on: http://localhost:5000\"\n- [ ] `http://localhost:5000/health` returns status\n- [ ] `http://localhost:5000/swagger` shows API documentation\n- [ ] No error messages in console\n\n### Frontend Running?\n\n- [ ] Terminal shows \"Angular Live Development Server is listening on localhost:4200\"\n- [ ] `http://localhost:4200` loads in browser\n- [ ] Page displays Naar & Noor homepage\n- [ ] No error messages in console\n\n### Apps Connected?\n\n- [ ] Frontend loads without CORS errors\n- [ ] API calls work (check browser Network tab)\n- [ ] Data displays on frontend (menu, chefs, etc.)\n- [ ] No 404 errors in console\n\n---\n\n## Testing the Connection\n\n### Test Backend API\n\n**Using cURL:**\n\n```bash\n# Get all chefs\ncurl http://localhost:5000/api/chefs\n\n# Get menu items\ncurl http://localhost:5000/api/menu\n\n# Health check\ncurl http://localhost:5000/health\n```\n\n**Using Browser:**\n\nOpen these URLs:\n- `http://localhost:5000/api/chefs`\n- `http://localhost:5000/api/menu`\n- `http://localhost:5000/swagger`\n\n### Test Frontend\n\n1. Open `http://localhost:4200` in browser\n2. Open Developer Tools (F12)\n3. Go to Network tab\n4. Refresh page\n5. Check that API calls succeed (200 status)\n\n---\n\n## Common Issues & Solutions\n\n### Backend Won't Start\n\n**Error:** `Address already in use`\n\n**Solution:**\n```bash\n# Use different port\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj --urls \"http://localhost:5001\"\n```\n\n**Error:** `Cannot open database \"NaarNoor\"`\n\n**Solution:**\n```bash\n# Create database\ndotnet ef database create --project src/NaarNoor.Infrastructure\n\n# Or apply migrations\ndotnet ef database update --project src/NaarNoor.Infrastructure\n```\n\n### Frontend Won't Start\n\n**Error:** `Port 4200 is already in use`\n\n**Solution:**\n```bash\nnpm run dev -- --port 4300\n```\n\n**Error:** `Cannot find module`\n\n**Solution:**\n```bash\nrm -rf node_modules package-lock.json\nnpm install\n```\n\n### CORS Error\n\n**Error:** `Access to XMLHttpRequest blocked by CORS policy`\n\n**Solution:**\n\nBackend CORS is already configured. If still getting error:\n\n1. Verify backend is running\n2. Check API URL in frontend service\n3. Restart both apps\n\n### API Calls Failing\n\n**Error:** `404 Not Found` or `Connection refused`\n\n**Solution:**\n\n1. Verify backend is running: `http://localhost:5000/health`\n2. Check API URL in `src/app/services/api.service.ts`\n3. Verify endpoint exists in backend\n4. Check browser Network tab for actual request\n\n---\n\n## Development Workflow\n\n### Making Changes\n\n**Backend Changes:**\n1. Edit C# files in `api-server/src/`\n2. Backend auto-reloads (if using `dotnet watch`)\n3. Or restart with Ctrl+C and rerun\n\n**Frontend Changes:**\n1. Edit TypeScript/HTML files in `naar-noor/src/`\n2. Frontend auto-reloads (hot reload enabled)\n3. Changes appear immediately in browser\n\n### Database Changes\n\n**Add Migration:**\n```bash\ncd api-server\ndotnet ef migrations add MigrationName --project src/NaarNoor.Infrastructure\n```\n\n**Apply Migration:**\n```bash\ndotnet ef database update --project src/NaarNoor.Infrastructure\n```\n\n---\n\n## Stopping the Apps\n\n### Stop Backend\n\nIn backend terminal:\n```bash\nCtrl + C\n```\n\n### Stop Frontend\n\nIn frontend terminal:\n```bash\nCtrl + C\n```\n\n### Stop Docker Compose\n\n```bash\ndocker-compose down\n```\n\n---\n\n## Performance Tips\n\n### Backend Performance\n\n- Use `AsNoTracking()` for read-only queries\n- Implement pagination for large datasets\n- Add database indexes\n- Use caching for frequently accessed data\n\n### Frontend Performance\n\n- Use lazy loading for routes\n- Optimize images\n- Minimize bundle size\n- Use OnPush change detection\n\n---\n\n## Debugging\n\n### Backend Debugging\n\n**Visual Studio:**\n1.
