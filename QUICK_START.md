# ⚡ Quick Start - Run Both Apps in 5 Minutes\n\n## 🎯 Goal\nGet both the Angular frontend and .NET backend running locally.\n\n---\n\n## ✅ Prerequisites (2 minutes)\n\nInstall these if you don't have them:\n\n### 1. Node.js 18+\n- Download: https://nodejs.org/\n- Verify: `node --version` (should be v18+)\n\n### 2. .NET 8.0 SDK\n- Download: https://dotnet.microsoft.com/download\n- Verify: `dotnet --version` (should be 8.0+)\n\n### 3. SQL Server\n- **Windows**: Use LocalDB (comes with Visual Studio)\n- **macOS/Linux**: Use Docker: `docker run -e 'ACCEPT_EULA=Y' -e 'SA_PASSWORD=YourPassword123!' -p 1433:1433 mcr.microsoft.com/mssql/server:2022-latest`\n\n---\n\n## 🚀 Start Both Apps (3 minutes)\n\n### Option A: Automatic (Easiest)\n\n**Windows:**\n```bash\nstart-dev.bat\n```\n\n**macOS/Linux:**\n```bash\nbash start-dev.sh\n```\n\nThis opens two terminal windows and starts both apps automatically.\n\n---\n\n### Option B: Manual (Two Terminal Windows)\n\n#### Terminal 1 - Backend\n\n```bash\ncd api-server\ndotnet restore\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj\n```\n\n**Wait for:**\n```\ninfo: Microsoft.Hosting.Lifetime[14]\n      Now listening on: http://localhost:5000\n```\n\n#### Terminal 2 - Frontend\n\n```bash\ncd naar-noor\nnpm install\nnpm run dev\n```\n\n**Wait for:**\n```\n✔ Angular Live Development Server is listening on localhost:4200\n```\n\n---\n\n## ✨ Verify Everything Works\n\n### Check Backend\n\nOpen in browser:\n- **Health Check**: http://localhost:5000/health\n- **Swagger API Docs**: http://localhost:5000/swagger\n\nYou should see JSON responses.\n\n### Check Frontend\n\nOpen in browser:\n- **Application**: http://localhost:4200\n\nYou should see the Naar & Noor homepage.\n\n### Check Connection\n\n1. Open http://localhost:4200\n2. Press F12 (Developer Tools)\n3. Go to Network tab\n4. Refresh page\n5. Look for API calls to `localhost:5000`\n6. They should have status 200 (green)\n\n---\n\n## 🎉 Success!\n\nBoth apps are running:\n\n| App | URL | Purpose |\n|-----|-----|----------|\n| Frontend | http://localhost:4200 | User interface |\n| Backend API | http://localhost:5000/api | Data & business logic |\n| API Docs | http://localhost:5000/swagger | API documentation |\n| Health Check | http://localhost:5000/health | Backend status |\n\n---\n\n## 🔧 Common Issues\n\n### \"Port 4200 already in use\"\n\n```bash\nnpm run dev -- --port 4300\n```\n\n### \"Port 5000 already in use\"\n\n```bash\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj --urls \"http://localhost:5001\"\n```\n\n### \"Cannot find module\"\n\n```bash\ncd naar-noor\nrm -rf node_modules package-lock.json\nnpm install\n```\n\n### \"Cannot open database\"\n\n```bash\ncd api-server\ndotnet ef database create --project src/NaarNoor.Infrastructure\n```\n\n### \"CORS error in browser\"\n\n1. Verify backend is running at http://localhost:5000/health\n2. Restart both apps\n3. Check browser console for actual error\n\n---\n\n## 📖 Next Steps\n\n1. ✅ Both apps running\n2. 📚 Read [Full Documentation](./docs/RUN_BOTH_APPS.md)\n3. 🏗️ Explore [Project Structure](./docs/PROJECT_STRUCTURE.md)\n4. 📡 Check [API Endpoints](./docs/API.md)\n5. 🔧 Review [Frontend Guide](./docs/FRONTEND.md)\n6. 💾 Review [Backend Guide](./docs/BACKEND.md)\n\n---\n\n## 🛑 Stop the Apps\n\n**In each terminal:**\n```bash\nCtrl + C\n```\n\n---\n\n## 💡 Tips\n\n- **Hot Reload**: Frontend auto-reloads when you save files\n- **Backend Changes**: Restart backend to see changes\n- **Database**: Migrations run automatically on startup\n- **Swagger**: Great for testing API endpoints\n- **DevTools**: Use F12 in browser to debug frontend\n\n---\n\n## 📞 Need Help?\n\nCheck [Troubleshooting Guide](./docs/TROUBLESHOOTING.md) or [Full Documentation](./docs/README.md)\n\n---\n\n**Happy coding! 🚀**\n
